Architecture

Fast, simple, and it just works.

All the complexity is hidden. You write data. You read data. At any scale. Without weekend on-call incidents.

PATH 01 / 03 — DURABLE WRITE

Value lands on NVMe before ack.

Group-commit WAL batches concurrent writes into a single fsync. The value goes straight to the VLog via O_DIRECT — the index keeps only a 24-byte pointer. Zero write amplification.

01App → WAL~12µs
02WAL fsync (group-commit)~80µs
03VLog → NVMe O_DIRECT~30µs
04Index pointer update~8µs
05Cache warm → ack~5µs
End-to-end · P99 ~0.2 ms
PATH 02 / 03 — CACHE READ

Hot key served in microseconds.

The LIRS cache learns your access pattern. Small hot keys (≤256 B) get priority-2 eviction resistance — they stay warm even during large cold scans. Zero disk I/O on hit.

01App → LIRS cache lookup~20µs
02LIR stack hit (in-memory)~15µs
03Zero-copy value return~5µs
04TCP response → ack~10µs
End-to-end · P99 ~0.3 ms
PATH 03 / 03 — TOMBSTONE DELETE

Key marked dead, instantly durable.

Delete follows the same group-commit WAL path as a write. A tombstone is stamped in the index shard. The VLog value is marked dead for the next GC pass — no immediate disk rewrite.

01App → WAL~12µs
02WAL fsync (group-commit)~80µs
03Tombstone → index shard~5µs
04Bloom filter update~2µs
05VLog GC marks dead~3µs
End-to-end · P99 ~0.2 ms
